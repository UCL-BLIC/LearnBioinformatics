Learn Bioinformatics

At the BLIC, we perform bioinformatics analyses for other groups at the UCL Cancer Institute. ​However, there is a growing trend, wherein an increasing number of wet-lab researchers are expressing an interest in conducting their own bioinformatics analyses, and they come to the BLIC for help and support. ​ ​In response to this demand, we have created this webpage to provide researchers from the CI, even those lacking prior computational expertise, with resources to gain the knowledge and skills required to proficiently analyze their own data.

But before you start, there are two main things we want to point out:

We don't want to reinvent the wheel, and therefore, this material is mainly a compilation of tutorials, courses and other pre-existing resources that are already available, and that explain some of the concepts that ​we believe you need to know to be able to perform bioinformatics data analyses confidently.

If you are unable to dedicate the necessary time and effort to thoroughly learn and practice the materials covered in these courses, we recommend refraining from attempting your own data analysis. Just as one cannot acquire the expertise of a wet lab scientist by merely reading about wet lab techniques, performing proper bioinformatic data analyses demands an investment of time, commitment, and a lot practice.

As an overview, here we list some programming skills that are commonly required for bioinformaticians:

  • ❯
    Cluster usage: ​Knowing how to use a cluster before conducting data analysis is vital for efficiency, scalability, and the successful execution of resource-intensive tasks. It enables researchers to make the most of available computational resources and streamline their analyses, ultimately leading to more reliable and timely results.
  • ❯
    Bash/Shell Scripting: Knowledge of shell scripting is important for automating routine tasks, running bioinformatics pipelines, and working with command-line tools.
  • ❯
    R: R is a widely used programming language in bioinformatics for statistical analysis, data manipulation, and data visualization. Proficiency in R allows for efficient data exploration and analysis.
  • ❯
    Python (Perl, or other programming language): Python is a versatile programming language that is extensively used in bioinformatics for various tasks such as data processing, scripting, and algorithm development. It provides a rich ecosystem of libraries and frameworks for bioinformatics analysis.
  • ❯
    Version Control Systems: Experience with version control systems like Git allows for efficient collaboration, code management, and tracking changes in projects.
  • ❯
    Next-Generation Sequencing (NGS) Analysis Tools: Familiarity with bioinformatics tools and libraries specific to NGS data analysis, such as samtools, BEDTools, or GATK, is highly valuable in processing and analyzing genomic sequencing data.
  • ❯
    Data Visualization: Proficiency in libraries like ggplot2 (R) or matplotlib (Python) for creating visualizations helps in presenting and interpreting complex biological data.

We have divided the material into three sections, ranging from fundamental concepts to more intricate data analyses.

If you intend to complete a comprehensive analysis from start to finish, we highly recommend mastering the content presented in the Fundamentals section before delving into more advanced topics. However, this prerequisite is not essential if your goal is limited to conducting tertiary analysis tasks. For instance:

  •  If you are given VCF files with variants, and just want to create oncoplot summarizing the results, then you can focus on the Basic R section and skip the rest. You can do this on your own laptop.
  • If you already have a matrix of gene counts, and just want to do some DGE analysis, you will want to just focus on the Basic R section and then move to the NGS (RNAseq) section. Again, you should be able to run this on your local machine without needing access to the UCL Cluster
